## 2. Контекст / Проблема

- «AI-бот» — не один агент: у кожної платформи свій user agent і своя мета crawling.
- Заблокований бот не вилучить контент, але robots.txt — це контроль доступу, а не видалення сторінки з Search.
- Рішення щодо відкритих корпусів (Common Crawl) — окреме від доступу для «живих» ботів і часто ухвалюється неявно.

## 3. Ядро

### 3.1. Карта агентів замість одного правила

OpenAI документує власні crawlers і user agents: GPTBot (тренувальний crawler) і OAI-SearchBot (для search-сценаріїв) [Джерело 70]. Anthropic описує ClaudeBot, Claude-User і Claude-SearchBot — окремий search crawler; його блокування може знизити видимість сайту в Claude search-подібних сценаріях [Джерело 75]. У Perplexity розрізняються PerplexityBot (surfacing/linking, керується robots.txt) і Perplexity-User (user-requested fetch, який зазвичай ігнорує robots.txt) — політику для них перевіряють окремо [Джерело 74]. Agent Traffic будує єдину карту цих агентів і показує політику по кожному окремо.

### 3.2. Що насправді робить robots.txt

Огляд Google crawlers and fetchers пояснює, які агенти Google використовують для різних задач [Джерело 76]. robots.txt керує доступом краулера до URL, але не утримує сторінку поза індексом — заблокований URL може зʼявитися без опису. Agent Traffic явно розділяє «контроль доступу» і «виключення з Search», щоб рішення ухвалювалися свідомо.

### 3.3. Common Crawl — окреме стратегічне рішення

Common Crawl документує CCBot — інфраструктуру відкритих веб-даних, на яку спираються багато AI-систем [Джерело 77]. Блокування «живих» ботів не дорівнює відсутності в цих корпусах. Agent Traffic виносить рішення щодо CCBot в окремий, явний контур.

### 3.4. Дрейф політики як джерело втрат

Політика доступу змінюється з релізами, CDN-правилами й WAF. Agent Traffic відстежує невідповідності між наміром (хочемо бути цитованими) і фактичним станом robots.txt/firewall для кожного агента з документації платформ [Джерело 70, 74, 75].

## 5. AEO / FAQ

### Чи можна сховати сторінку від Search через Agent Traffic?
Ні. Agent Traffic керує доступом краулера, але robots.txt не видаляє URL зі Search — для цього потрібні `noindex` або password protection [Джерело 76].

### Навіщо окремо керувати CCBot?
Бо блокування «живих» ботів не прибирає контент із відкритих корпусів Common Crawl, якими користуються інші системи; це окреме рішення [Джерело 77].

### Чи достатньо одного правила «для AI-ботів»?
Ні. GPTBot, ClaudeBot, PerplexityBot і Googlebot — різні агенти з різними цілями; політику задають по кожному окремо [Джерело 70, 74, 75].

### З чого почати?
З інвентаризації фактичного robots.txt і firewall проти документації OpenAI, Perplexity, Anthropic, Google і Common Crawl — це найдешевша перевірка GEO-аудиту [Джерело 76, 74].
