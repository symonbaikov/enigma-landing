## 2. Контекст / Проблема (pain points)

- AI-відповідь може заміняти частину класичного SERP-шляху для товарних і порівняльних запитів.
- Товарна картка, написана лише для людини, погано вилучається як самодостатній пасаж.
- Органічний трафик чутливий до структури й релевантності контенту — а AI-канал додає окремий рівень видимості зверху.

## 3. Ядро (benefits)

### 3.1. E-commerce — окреме поле GEO

Bagga et al. (E-GEO) — тестова середовище для GEO саме в e-commerce: видимість товарів, релевантність і взаємодія товарних сторінок із генеративними відповідями [Джерело 04]. Це підтверджує: товарний контент потребує окремого підходу, а не загальних SEO-правил.

### 3.2. Що рухає органічний трафик ритейлу

Baye, De los Santos, Wildenbeest досліджують, які фактори рухають органічний трафик retail-сайтів [Джерело 59]. GEO не скасовує цю базу: технічна доступність і релевантність лишаються передумовою, поверх якої додається видимість у AI-відповіді.

### 3.3. AI Overviews зачіпають товарні запити

Дослідження Semrush по AI Overviews дає ринкову картину того, які типи запитів і сайтів частіше зачіпає AI-відповідь [Джерело 80]. Для e-commerce це сигнал: на частині товарних/порівняльних запитів класичний клік зменшується, тож потрібна присутність у самій відповіді.

### 3.4. Що з цього випливає

Товарний пасаж має бути самодостатнім: чітка назва, ключові атрибути, призначення, обмеження — придатний для вилучення як відповідь на конкретне товарне питання. Це редакційна евристика зниження ризику, не гарантія inclusion. Окремий офіційний контур товарної видимості в Google — Merchant Center, product feed і Google Business Profile: за документацією Google, AI-відповіді можуть включати product listings, а Merchant Center допомагає такій видимості [Джерело 65].

| Тип запиту | Класичний шлях | GEO-підхід для e-commerce |
|------------|----------------|----------------------------|
| «найкращий [категорія] для X» | Категорійна сторінка в топі | Товарні пасажі, придатні для synthesised добірки |
| «[товар A] vs [товар B]» | Порівняльна стаття | Структуровані атрибути + зовнішні огляди |
| Транзакційний | PDP у видачі | Базове SEO + Merchant Center/product feed + AI-видимість зверху [65] |

## 5. AEO / FAQ

### Чи потрібен e-commerce окремий GEO-підхід?
Так. E-GEO показує, що видимість товарів і взаємодія товарних сторінок із генеративними відповідями — окреме поле, а не загальні SEO-правила [Джерело 04].

### GEO скасовує класичне SEO для магазину?
Ні. Фактори органічного трафику ритейлу лишаються передумовою; AI-видимість додається поверх, а не замість [Джерело 59].

### Які запити найбільш під загрозою?
Товарні й порівняльні, де AI-відповідь дає готову добірку; ринкові дані показують зростання покриття AI Overviews [Джерело 80].

### Як зробити товарну картку «придатною для AI»?
Самодостатній пасаж: чітка назва, ключові атрибути, призначення, обмеження — як відповідь на конкретне товарне питання (евристика, не гарантія inclusion); паралельно вести Merchant Center / product feed як офіційний Google-контур товарної видимості [Джерело 04, 65].

## 6. Schema (плейсхолдери)

> Псевдокод-плейсхолдери для ревʼю. У production — валідний JSON-LD без коментарів. **Увага:** FAQ rich results вилучені з Google Search із 07.05.2026 — `FAQPage` лише як опціональна семантична розмітка (не Google rich-result lever); основна схема — `Article` (для каталогу — `Product`/`Offer`). Видимий FAQ-контент зберігаємо. **У коді:** `Article` завжди; видимий FAQ завжди; `FAQPage` — опціонально / за feature-flag; коментарі `//` та `/* */` видалити, `mainEntity` заповнити.

```jsonld
// Article
{ "@context":"https://schema.org", "@type":"Article",
  "headline":"GEO для e-commerce",
  "about":["GEO for e-commerce","product visibility","AI Overviews"],
  "dateModified":"{{UPDATED_AT}}" }
```
```jsonld
// FAQPage — мапиться з блоку 5 (4 Q&A)
{ "@context":"https://schema.org", "@type":"FAQPage", "mainEntity":[ /* блок 5 */ ] }
```
