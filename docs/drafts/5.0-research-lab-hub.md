## 5. AEO / FAQ

### Що таке Research Lab?
Набір розборів наукових і галузевих джерел про AI-видимість: zero-click, перетин AI Overviews із SERP, нестабільність відповідей, citation bias — із явним маркуванням рівня доказовості.

### Чим це відрізняється від блогу агенції?
Кожен матеріал прив'язаний до первинного джерела з каталогу `docs/research.md` і розділяє доведені факти й редакційні евристики, а не подає обіцянки [Джерело 06, 39].

### З чого почати?
Зі статті 5.1 про zero-click — вона дає бізнес-контекст, чому видимість усередині відповіді стала окремою задачею [Джерело 83, 78].

### Чи це маркетингові кейси?
Ні. Це розбори досліджень; клієнтські метрики, якщо є, наводяться окремо й лише як реальні дані, без вигаданих цифр.

## 6. Schema (плейсхолдери)

> Псевдокод-плейсхолдери для ревʼю. У production — валідний JSON-LD без коментарів. **Увага:** FAQ rich results вилучені з Google Search із 07.05.2026 — `FAQPage` лише як опціональна семантична розмітка (не Google rich-result lever); основна схема — `Article` (для хаба — `CollectionPage`). Видимий FAQ-контент зберігаємо. **У коді:** `Article` завжди; видимий FAQ завжди; `FAQPage` — опціонально / за feature-flag; коментарі `//` та `/* */` видалити, `mainEntity` заповнити.

```jsonld
// Article — для сторінки-хаба
{ "@context":"https://schema.org", "@type":"Article",
  "headline":"Research Lab: дані про AI-видимість",
  "about":["zero-click search","AI Overviews","citation bias","AI visibility"],
  "dateModified":"{{UPDATED_AT}}" }
```
```jsonld
// FAQPage — мапиться з блоку 5 (4 Q&A)
{ "@context":"https://schema.org", "@type":"FAQPage", "mainEntity":[ /* блок 5 */ ] }
```
