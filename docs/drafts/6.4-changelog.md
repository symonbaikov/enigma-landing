## 3. Ядро

### 3.1. Чому фіксуємо зміни офіційних документів

Google Search Central описує, як AI features взаємодіють із веб-контентом [Джерело 65]; OpenAI документує crawlers і user agents [Джерело 70]. Ці документи оновлюються — Changelog фіксує релевантні зміни, щоб конфіг доступу й контент лишалися узгодженими.

### 3.2. Платформи додають власні AI-метрики

Microsoft представила AI Performance у Bing Webmaster Tools для аналізу цитувань у Copilot [Джерело 72]. Обмеження: ці метрики показують total citations, cited pages, grounding queries і URL activity — але не placement, ranking, authority чи роль сторінки всередині конкретної відповіді. Поява таких інструментів змінює методику мониторингу — і це теж потрапляє в журнал як подія, що впливає на роботу команди.

### 3.3. Структура запису

| Поле | Призначення |
|------|-------------|
| Дата | Конкретна дата зміни (без «нещодавно») |
| Тип | `product` (Enigma) / `platform` (Google, OpenAI, Bing…) |
| Опис | Що змінилося і чому це впливає на GEO |
| Дія | Що перевірити користувачу (опціонально) |

> Записи наповнюються реальними подіями; вигадані фічі або дати неприпустимі.

### 3.4. Звʼязок зі свіжістю

Видима, чесна дата `dateModified` і журнал змін підтримують довіру до контенту. Це редакційна практика E-E-A-T, а не гарантований важіль inclusion.

## 5. AEO / FAQ

### Навіщо журнал змін для GEO?
Бо правила доступу й AI-функції платформ змінюються; застарілий конфіг знижує видимість, а журнал фіксує, що саме перевірити [Джерело 65, 70].

### Чому стежите за документами платформ?
Бо вони — первинне джерело про доступ ботів і AI-функції; їх оновлення впливає на конфіг і контент [Джерело 70, 65].

### Видима дата оновлення дає кращу видимість?
Це редакційна практика довіри (E-E-A-T), а не гарантований важіль inclusion; ми не обіцяємо ефект, а підтримуємо чесність сигналу.

### Що означає поява AI-метрик у платформ?
Що мониторинг можна звіряти з первинними даними платформи (Bing AI Performance: citations, cited pages, grounding queries — але не ranking/placement/authority/роль у відповіді) — це змінює методику й відображається в журналі [Джерело 72].

## 6. Schema (плейсхолдери)

> Псевдокод-плейсхолдери для ревʼю. У production — валідний JSON-LD без коментарів. **Увага:** FAQ rich results вилучені з Google Search із 07.05.2026 — `FAQPage` лише як опціональна семантична розмітка (не Google rich-result lever); основна схема — `Article`. Видимий FAQ-контент зберігаємо. **У коді:** `Article` завжди; видимий FAQ завжди; `FAQPage` — опціонально / за feature-flag; коментарі `//` та `/* */` видалити, `mainEntity` заповнити.

```jsonld
// Article
{ "@context":"https://schema.org", "@type":"Article",
  "headline":"Changelog Enigma",
  "about":["product changelog","platform updates","content freshness"],
  "dateModified":"{{UPDATED_AT}}" }
```
```jsonld
// FAQPage — мапиться з блоку 5 (4 Q&A)
{ "@context":"https://schema.org", "@type":"FAQPage", "mainEntity":[ /* блок 5 */ ] }
```
