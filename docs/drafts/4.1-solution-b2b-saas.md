## 2. Контекст / Проблема (pain points)

- AI-відповідь часто синтезується з зовнішніх джерел і незалежних згадок, а не лише з вашого сайту.
- Складні порівняльні й explanatory-запити — саме той сценарій, де користувач обирає готову AI-відповідь замість списку посилань.
- Zero-click: значна частина запитів не доходить до відкритого вебу — лід не бачить вашого сайту, навіть якщо ви в топі.

## 3. Ядро (benefits)

### 3.1. Видимість поза власним сайтом

Chen, Wang, Chen, Koudas показують систематичне зміщення AI Search у бік earned media й незалежних джерел [Джерело 02]. Для B2B SaaS це означає: GEO-стратегія керує ширшим контуром — каталоги, огляди, порівняння, медіа, — а не лише сторінками продукту.

### 3.2. AI виграє саме на складних порівняннях

Caramancion досліджує, коли користувачі віддають перевагу LLM-відповіді, а коли класичному пошуку: готова відповідь виграє на складних, порівняльних і explanatory задачах [Джерело 15]. Для SaaS це ядро воронки — «X vs Y», «найкраще рішення для Z», «як обрати».

### 3.3. Zero-click зменшує верх воронки

SparkToro / Datos: значна частка Google-пошуків не дає кліку на відкритий веб [Джерело 83]. Pew Research: за наявності AI summary користувачі рідше клікають по посиланнях [Джерело 78]. Це ризик-рамка, а не універсальний закон для всіх B2B-покупців: zero-click дослідження — panel/clickstream-вибірка, не повний лог усіх пошуків. Висновок для B2B: для частини покупців цінність зміщується з кліку на присутність усередині відповіді на порівняльний запит.

### 3.4. Що з цього випливає для стратегії

Робота над власними сторінками лишається передумовою, але недостатня: треба системно зʼявлятися в незалежних джерелах, які AI використовує як підтвердження конкретного твердження про вашу категорію [Джерело 02, 15].

| Сценарій B2B | Класичний SEO | GEO-підхід |
|--------------|---------------|------------|
| «X vs Y» | Сторінка порівняння в топі | Присутність у синтезованій відповіді + earned media |
| «найкраще для [індустрія]» | Лендинг рішення | Згадка в незалежних оглядах і каталогах |
| Технічна оцінка | Документація на сайті | Самодостатні пасажі, придатні для вилучення |

## 5. AEO / FAQ

### Чому топ-1 у Google не рятує B2B-лійку?
Бо на складних порівняльних запитах користувач часто бере готову AI-відповідь без кліку; позиція в SERP і присутність у відповіді можуть розходитися [Джерело 78, 15].

### Достатньо оптимізувати сторінки продукту?
Ні. AI Search надає вагу earned media й незалежним джерелам; стратегія має охоплювати зовнішній контур категорії [Джерело 02].

### Де GEO дає максимальний ефект для SaaS?
На explanatory й порівняльних запитах верху воронки — саме там користувач віддає перевагу синтезованій відповіді [Джерело 15].

### Що таке zero-click для B2B?
Сценарій, коли запит не доходить до відкритого вебу; лід не бачить сайт, тож потрібна присутність усередині відповіді. Цифри — panel/clickstream-оцінка, рамка ринку, а не абсолютна статистика всіх вертикалей [Джерело 83].

## 6. Schema (плейсхолдери)

> Псевдокод-плейсхолдери для ревʼю. У production — валідний JSON-LD без коментарів. **Увага:** FAQ rich results вилучені з Google Search із 07.05.2026 — `FAQPage` лише як опціональна семантична розмітка (не Google rich-result lever); основна схема — `Article`. Видимий FAQ-контент зберігаємо. **У коді:** `Article` завжди; видимий FAQ завжди; `FAQPage` — опціонально / за feature-flag; коментарі `//` та `/* */` видалити, `mainEntity` заповнити.

```jsonld
// Article
{ "@context":"https://schema.org", "@type":"Article",
  "headline":"GEO для B2B SaaS",
  "about":["GEO for SaaS","earned media","zero-click search"],
  "dateModified":"{{UPDATED_AT}}" }
```
```jsonld
// FAQPage — мапиться з блоку 5 (4 Q&A)
{ "@context":"https://schema.org", "@type":"FAQPage", "mainEntity":[ /* блок 5 */ ] }
```
